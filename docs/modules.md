# CA Role Module Reference

This document describes the custom Ansible modules in `roles/ca/library`.

The module documentation style follows the current Ansible guidance: describe
what each module and option does, keep defaults aligned with the code, identify
conditionally required values, document secret parameters, and state check mode
support. These modules are role-local modules, so this file is the human-facing
reference instead of generated `ansible-doc` pages.

## Common Behavior

- The modules use Python `cryptography` instead of the OpenSSL command line,
  except for no-op file assembly modules that only concatenate existing files.
- File-producing modules are idempotent by comparing existing files with the
  desired key, certificate, CRL, chain, bundle, or DH parameter content.
- `ca_fritzbox_deploy` compares the desired certificate with the current
  FRITZ!Box HTTPS certificate and uploads only when they differ, unless
  `force=true`.
- All modules currently set `supports_check_mode=False`.
- `force: true` makes the module regenerate or rewrite managed artifacts.
- File writes use an atomic temporary file and refuse to read, chmod, or replace
  a final symlink target through the shared `ca_file` helper.
- Secret-looking values are masked from module failure messages by the shared
  error sanitizer. Parameters marked as secrets in the module spec also use
  `no_log=True`.
- Unless explicitly configured otherwise, certificate and CRL signatures use
  `sha384`. Ed25519 and Ed448 private keys sign without a digest, as required
  by EdDSA.

## Path Layout

The CA modules derive paths from `base_dir` and `name`.

| Artifact | Authority path | Certificate path |
| --- | --- | --- |
| Private key | `<base_dir>/private/<name>-ca.key` | `<output_dir>/<name>.key` |
| CSR | `<base_dir>/csr/<name>-ca.csr` | `<base_dir>/csr/<name>.csr` |
| PEM certificate | `<base_dir>/ca/<name>-ca.pem` | `<output_dir>/<name>.pem` |
| DER certificate | `<base_dir>/ca/<name>-ca.der` | `<output_dir>/<name>.der` |
| Text certificate | `<base_dir>/ca/<name>-ca.txt` | `<output_dir>/<name>.txt` |
| Chain | `<base_dir>/chains/<name>-ca-chain.pem` | `<output_dir>/<name>-chain.pem` |
| Fullchain bundle | Not generated for authorities | `<output_dir>/<name>-fullchain.pem` |
| FritzBox bundle | Not generated for authorities | `<output_dir>/<name>-fritzbox.pem` |
| CRL | `<base_dir>/crl/<name>-ca.crl.pem` or `<base_dir>/crl/<name>-ca.crl` | Not generated by certificate modules |

For certificate modules, `output_dir` defaults to
`<base_dir>/certs/<name>`.

## Common Authority Parameters

These parameters are used by `ca_authority`.

| Parameter | Type | Required | Default | Secret | Behavior |
| --- | --- | --- | --- | --- | --- |
| `base_dir` | path | yes | none | no | Base directory for CA private keys, CSRs, certificates, chains, and CRLs. |
| `base_url` | str | no | `""` | no | Base publication URL. If set, AIA defaults to `<base_url>/aia/...` and CDP defaults to `<base_url>/crl/...`. |
| `name` | str | yes | none | no | Authority short name. It is used in all derived file names. |
| `formats` | list[str] | no | `["pem", "der", "txt"]` | no | Certificate output formats. PEM is always written; DER and text exports are written when `der` or `txt` are present. |
| `key_type` | str | no | `RSA` | no | Private key algorithm. Supported values include `RSA`, `ECDSA`, `EC`, `P256`, `P384`, `ECDSA_P256`, `ECDSA_P384`, `Ed25519`, and `Ed448`. |
| `key_size` | int | no | `4096` | no | RSA key size in bits, or ECDSA curve selector `256` or `384`. Ignored for Ed25519 and Ed448. |
| `key_passphrase` | str | yes | none | yes | Passphrase used to encrypt the authority private key. |
| `subject_ordered` | list[dict] | no | `[]` | no | Ordered subject override. When set, it fully defines the subject and takes precedence over `common_name`, `email`, and `subject`. |
| `common_name` | str | conditional | none | no | Common Name used when `subject_ordered` is not set. |
| `email` | str | no | none | no | Adds `emailAddress` to the generated subject when `subject_ordered` is not set. |
| `subject` | dict | no | `{}` | no | Subject defaults. Supported semantic keys are `country`, `state`, `locality`, `organization`, and `organizational_unit`; short keys `C`, `ST`, `L`, `O`, and `OU` are also accepted. |
| `basic_constraints` | list[str] | no | module profile | no | OpenSSL-like Basic Constraints tokens. Authority modules set CA defaults. |
| `key_usage` | list[str] | no | module profile | no | Key Usage tokens. Authority modules default to `keyCertSign` and `cRLSign`. |
| `key_usage_critical` | bool | no | `true` | no | Marks Key Usage critical. |
| `extended_key_usage` | list[str] | no | `[]` | no | EKU names or dotted OIDs. |
| `extended_key_usage_critical` | bool | no | `false` | no | Marks Extended Key Usage critical. |
| `san` | list[str] | no | `[]` | no | SAN entries such as `DNS:name`, `IP:192.0.2.10`, `email:user@example.test`, `URI:https://...`, `RID:1.2.3`, or `otherName:<oid>;UTF8:<value>`. |
| `san_critical` | bool | no | `false` | no | Marks SAN critical. |
| `aia_base_url` | str | no | `""` | no | Explicit AIA base URL. The module appends `<name>-ca.der`. |
| `cdp_base_url` | str | no | `""` | no | Explicit CDP base URL. The module appends `<name>-ca.crl`. |
| `raw_extensions` | list[dict] | no | `[]` | no | Additional unrecognized extensions with `oid`, `value`, and optional `critical`. |
| `pkinit` | dict | no | `{}` | no | Internal PKINIT context for SAN otherName encoding. Normally only the `mskdc` certificate profile sets this. |
| `days` | int | yes | none | no | Certificate validity in days. |
| `digest` | str | no | `sha384` | no | Signature digest for RSA and ECDSA signers. Supported digests are `sha1`, `sha224`, `sha256`, `sha384`, and `sha512`. |
| `include_identifiers` | bool | no | `true` | no | Adds Subject Key Identifier and Authority Key Identifier. |
| `owner` | str | no | none | no | Owner applied to generated files. When omitted, ownership is left unchanged. |
| `group` | str | no | none | no | Group applied to generated files. When omitted, group ownership is left unchanged. |
| `key_mode` | str | no | `0600` | no | Private key file mode. |
| `public_mode` | str | no | `0644` | no | CSR, certificate, DER, text, and chain file mode. |
| `force` | bool | no | `false` | no | Regenerates managed material even if existing files match. |

Supported `subject_ordered` keys are `C`, `countryName`, `ST`,
`stateOrProvinceName`, `L`, `localityName`, `O`, `organizationName`, `OU`,
`organizationalUnitName`, `CN`, `commonName`, and `emailAddress`.

Supported `raw_extensions.value` encodings are `ASN1:BMPSTRING:<value>`,
`ASN1:UTF8String:<value>`, `ASN1:FORMAT:HEX,OCTETSTRING:<hex>`, and `DER:<hex>`.

The `certificate` dictionary can also carry advanced X.509 model keys that are
not explicit top-level module arguments: `digest`, `basic_constraints`,
`key_usage_critical`, `extended_key_usage_critical`, `san_critical`, `pkinit`,
`include_identifiers`, `key_mode`, `public_mode`, and `directory_mode`.

`ca_certificate` creates the output directory, generates or reuses the private
key, write a CSR under `<base_dir>/csr`, issue the certificate with the issuer
CA, optionally write DER and text exports, and copy the issuer chain into the
certificate directory.

## `ca_certificate`

Dispatches a declarative role certificate to the built-in certificate profiles.

This is the module used by the role task for entries in `ca_certificates`. It is
not a generic X.509 primitive; it dispatches the declarative certificate type to
built-in role profiles.

Behavior:

- Validates certificate `name`, `type`, `common_name`, and profile-specific
  required fields.
- Resolves the issuer through `certificate_types`.
- Reads issuer passphrase and `default_days` from the matching authority in
  `authorities`.
- Merges the role-level `subject` dictionary with any certificate-local
  `subject` dictionary.
- Applies built-in profile defaults for formats, key usage, EKU, SAN defaults,
  raw extensions, and digest.
- Generates certificate material through the shared X.509 helper and returns
  export hints for PKCS#12, fullchain, and FritzBox bundle tasks.

Parameters:

| Parameter | Type | Required | Default | Secret | Behavior |
| --- | --- | --- | --- | --- | --- |
| `base_dir` | path | yes | none | no | Base CA directory used to locate issuer material and derive output paths. |
| `base_url` | str | no | `""` | no | Base publication URL. If set, AIA defaults to `<base_url>/aia/...` and CDP defaults to `<base_url>/crl/...`. |
| `certificate` | dict | yes | none | yes | One declarative `ca_certificates` item. Must contain at least `name`, `type`, and `common_name`. |
| `certificate_types` | dict | yes | none | no | Certificate type map. The selected type must define `issuer` and may define `required_fields`. |
| `authorities` | list[dict] | yes | none | yes | Authority list used to resolve the issuer passphrase and issuer `default_days`. |
| `kerberos_realm` | str | no | `""` | no | Default realm for `mskdc` certificates when the certificate does not set `krb5_realm`. |
| `subject` | dict | no | `{}` | no | Role-level subject defaults merged with certificate-local subject values. |
| `owner` | str | no | none | no | Owner applied to generated files. When omitted, ownership is left unchanged. |
| `group` | str | no | none | no | Group applied to generated files. When omitted, group ownership is left unchanged. |
| `force` | bool | no | `false` | no | Regenerates managed material even if existing files match. |

The `certificate` dictionary may carry the same certificate model keys accepted
by the concrete X.509 modules, including `formats`, `output_dir`, key settings,
SANs, subject data, extension settings, `pfx_passphrase`, and profile-specific
fields such as `ad_object_guid` for `mskdc`.

Returns:

- `changed`
- `directory_changed`
- `key_changed`
- `csr_changed`
- `cert_changed`
- `der_changed`
- `txt_changed`
- `chain_changed`
- `formats`
- `pkcs12_formats`
- `fullchain_bundle`
- `fritzbox_bundle`
- `csr_path`
- `cert_path`
- `txt_path`
- `name`
- `profile`

## `ca_authority`

Manages a CA authority certificate.

Behavior:

- Generates an encrypted private key, CSR, PEM certificate, and optional DER
  and text exports for a CA authority.
- Uses `parent == name` or an omitted `parent` as the self-signed Root CA mode.
- Uses `parent != name` as the Issuing CA mode and signs the authority
  certificate with the parent CA.
- Defaults Root CA `basic_constraints` to `CA:TRUE,pathlen:1`.
- Defaults Issuing CA `basic_constraints` to `CA:TRUE,pathlen:0`.
- Defaults `key_usage` to `keyCertSign,cRLSign`.
- Embeds AIA and CDP URLs when `base_url`, `aia_base_url`, or `cdp_base_url`
  are provided.

Parameters:

- Accepts all common authority parameters.
- `parent` defaults to `name`.
- `parent_key_passphrase` is required when `parent != name`.
- `key_passphrase` is required.
- `common_name` is required unless `subject_ordered` is supplied.

Returns:

- `changed`
- `directory_changed`
- `key_changed`
- `csr_changed`
- `cert_changed`
- `der_changed`
- `txt_changed`
- `chain_changed`
- `formats`
- `csr_path`
- `cert_path`
- `txt_path`

## Built-In Certificate Profiles

`ca_certificate` supports these `type` values:

- `tls_server`: Defaults `formats` to `pem,der,txt`, adds DNS SAN from
  `common_name`, Key Usage `digitalSignature,keyEncipherment`, and EKU
  `serverAuth`.
- `tls_client`: Defaults `formats` to `pem,der,txt`, Key Usage
  `digitalSignature,keyEncipherment`, and EKU `clientAuth`.
- `eap_tls_client`: Defaults `formats` to `pem,der,txt`, Key Usage
  `digitalSignature,keyEncipherment`, and EKU `clientAuth`.
- `identity`: Defaults `formats` to `pem,der,txt,pfx`, Key Usage
  `digitalSignature,keyEncipherment,nonRepudiation`, and EKUs `clientAuth`,
  `emailProtection`, and smartcard logon OID `1.3.6.1.4.1.311.20.2.2`.
- `identity_full`: Uses the `identity` defaults and adds `codeSigning`.
- `mskdc`: Defaults `formats` to `pem,der,txt`, DNS SAN from `common_name`,
  Key Usage `digitalSignature,keyEncipherment`, EKUs `serverAuth`,
  `clientAuth`, and KDC Authentication OID `1.3.6.1.5.2.3.5`. It also adds
  Microsoft certificate template name OID `1.3.6.1.4.1.311.20.2` with
  BMPString value `DomainController`, NTDS objectGUID OID
  `1.3.6.1.4.1.311.25.1`, and PKINIT SAN otherName OID `1.3.6.1.5.2.2`.
- `fritzbox`: Defaults `formats` to `pem,der,txt,fritzbox`, DNS SAN from
  `common_name`, Key Usage `digitalSignature,keyEncipherment`, EKUs
  `serverAuth,clientAuth`, and `CA:FALSE` Basic Constraints. The configured
  digest is limited to `sha1`, `sha224`, `sha256`, or `sha384`.

Profile-specific certificate fields:

| Type | Required fields | Notes |
| --- | --- | --- |
| `mskdc` | `ad_object_guid`; `krb5_realm` unless global `ca_kerberos_realm` is set | `krb5_realm` is uppercased and encoded as `krbtgt/<REALM>@<REALM>`. |
| `identity`, `identity_full` | `pfx_passphrase` when the effective formats include `pfx` or `p12` | PKCS#12 export is handled by `ca_pkcs12_bundle`. |
| `fritzbox` | none beyond common certificate fields | FritzBox PEM export is handled by `ca_fritzbox_bundle`. |

Add `fullchain` to the effective `formats` list to request
`ca_fullchain_bundle` output.

## `ca_chain`

Manages an ordered PEM CA chain file.

Behavior:

- Reads `<base_dir>/ca/<name>-ca.pem`.
- If `parent` is set and differs from `name`, appends
  `<base_dir>/chains/<parent>-ca-chain.pem`.
- Normalizes every source certificate to PEM and writes
  `<base_dir>/chains/<name>-ca-chain.pem`.

Parameters:

| Parameter | Type | Required | Default | Secret | Behavior |
| --- | --- | --- | --- | --- | --- |
| `base_dir` | path | yes | none | no | Base CA directory. |
| `name` | str | yes | none | no | Authority short name for the output chain. |
| `parent` | str | no | `""` | no | Parent authority short name. Set it to `name` for a self-signed root. |
| `owner` | str | no | none | no | Owner applied to the chain file. |
| `group` | str | no | none | no | Group applied to the chain file. |
| `mode` | str | no | `0644` | no | Chain file mode. |
| `force` | bool | no | `false` | no | Rewrites the chain even if content matches. |

Returns:

- `changed`
- `path`

## `ca_crl`

Manages a certificate revocation list for a CA.

Behavior:

- Writes PEM CRLs to `<base_dir>/crl/<name>-ca.crl.pem`.
- Writes DER CRLs to `<base_dir>/crl/<name>-ca.crl`.
- Reads the CA private key from `<base_dir>/private/<name>-ca.key`.
- Signs the CRL with the CA private key.
- Regenerates existing CRLs when the issuer, signature algorithm, revocation
  entries, or `nextUpdate` validity no longer match the requested state.
- Compares issuer and revoked certificate serial/reason pairs for idempotence.

Parameters:

| Parameter | Type | Required | Default | Secret | Behavior |
| --- | --- | --- | --- | --- | --- |
| `base_dir` | path | yes | none | no | Base CA directory. |
| `name` | str | yes | none | no | Authority short name. |
| `format` | str | no | `pem` | no | CRL format. Choices are `pem` and `der`. |
| `key_passphrase` | str | yes | none | yes | Passphrase for the CA private key. |
| `common_name` | str | yes | none | no | Issuer subject common name. |
| `subject` | dict | no | `{}` | no | Issuer subject defaults. Uses the same keys as the common X.509 parameters. |
| `next_update_days` | int | yes | none | no | Number of days until the CRL `nextUpdate` timestamp. |
| `revoked_certificates` | list[dict] | no | `[]` | no | Revoked certificate entries. |
| `digest` | str | no | `sha384` | no | Signature digest for RSA and ECDSA CA keys. |
| `owner` | str | no | none | no | Owner applied to the CRL file. |
| `group` | str | no | none | no | Group applied to the CRL file. |
| `mode` | str | no | `0644` | no | CRL file mode. |
| `force` | bool | no | `false` | no | Rewrites the CRL even if existing issuer, signature algorithm, revocation entries, and `nextUpdate` validity match. |

`revoked_certificates` entries support:

| Key | Required | Behavior |
| --- | --- | --- |
| `serial_number` or `serial` | yes | Decimal, `0x` hex, or colon-separated hex serial number. |
| `reason` | no | One of `key_compromise`, `ca_compromise`, `affiliation_changed`, `superseded`, `cessation_of_operation`, `certificate_hold`, `privilege_withdrawn`, or `aa_compromise`. |
| `revocation_date` | no | ISO timestamp or ASN.1 GeneralizedTime-style `YYYYMMDDHHMMSSZ`. Defaults to current UTC time. |

Returns:

- `changed`

## `ca_pkcs12_bundle`

Manages a PKCS#12 bundle for a certificate.

Behavior:

- Reads the private key from `<output_dir>/<name>.key`.
- Reads the leaf certificate from `<output_dir>/<name>.pem`.
- Reads additional certificates from `<output_dir>/<name>-chain.pem`.
- Writes `<output_dir>/<name>.pfx` or `<output_dir>/<name>.p12`.
- Compares existing bundle key, leaf certificate fingerprint, and extra
  certificate fingerprints for idempotence.
- Requires an export passphrase.

Parameters:

| Parameter | Type | Required | Default | Secret | Behavior |
| --- | --- | --- | --- | --- | --- |
| `base_dir` | path | yes | none | no | Base CA directory. Used to derive default `output_dir`. |
| `certificate` | dict | no | `{}` | yes | Certificate model dictionary. May provide `output_dir`, `key_passphrase`, `pfx_passphrase`, `passphrase`, `friendly_name`, and `common_name`. |
| `name` | str | yes | none | no | Certificate short name and output file stem. |
| `output_dir` | path | no | `<base_dir>/certs/<name>` | no | Directory containing key, certificate, and chain files. |
| `format` | str | yes | none | no | Bundle format. Choices are `pfx` and `p12`. |
| `friendly_name` | str | no | certificate `common_name` or `name` | no | Friendly name stored in the PKCS#12 bundle. |
| `key_passphrase` | str | no | none | yes | Passphrase for the input private key. |
| `passphrase` | str | conditional | `certificate.pfx_passphrase` | yes | Export passphrase. Required unless provided by `certificate.pfx_passphrase`. |
| `owner` | str | no | none | no | Owner applied to the bundle file. |
| `group` | str | no | none | no | Group applied to the bundle file. |
| `mode` | str | no | `0600` | no | Bundle file mode. |
| `force` | bool | no | `false` | no | Rewrites the bundle even if content matches. |

Returns:

- `changed`
- `path`

## `ca_fullchain_bundle`

Assembles a normal PEM fullchain bundle from existing files.

Behavior:

- Reads source files from `<output_dir>/<name>.pem` and
  `<output_dir>/<name>-chain.pem`.
- Writes `<output_dir>/<name>-fullchain.pem`.
- The module concatenates normalized file content. It does not parse or validate
  the PEM objects.

Parameters:

| Parameter | Type | Required | Default | Secret | Behavior |
| --- | --- | --- | --- | --- | --- |
| `base_dir` | path | yes | none | no | Base CA directory. Used to derive default `output_dir`. |
| `certificate` | dict | no | `{}` | yes | Certificate model dictionary. May provide `output_dir`. |
| `name` | str | yes | none | no | Certificate short name and output file stem. |
| `output_dir` | path | no | `<base_dir>/certs/<name>` | no | Directory containing source files and receiving the bundle. |
| `owner` | str | no | none | no | Owner applied to the bundle file. |
| `group` | str | no | none | no | Group applied to the bundle file. |
| `mode` | str | no | `0644` | no | Bundle file mode. |
| `force` | bool | no | `false` | no | Rewrites the bundle even if content matches. |

Returns:

- `changed`
- `path`

## `ca_fritzbox_bundle`

Assembles a FritzBox PEM import bundle from existing files.

Behavior:

- Uses the fixed FritzBox import order: certificate, chain, private key.
- Reads source files from `<output_dir>/<name>.pem`,
  `<output_dir>/<name>-chain.pem`, and `<output_dir>/<name>.key`.
- Writes `<output_dir>/<name>-fritzbox.pem`.
- The module concatenates normalized file content. It does not parse or validate
  the PEM objects.

Parameters:

| Parameter | Type | Required | Default | Secret | Behavior |
| --- | --- | --- | --- | --- | --- |
| `base_dir` | path | yes | none | no | Base CA directory. Used to derive default `output_dir`. |
| `certificate` | dict | no | `{}` | yes | Certificate model dictionary. May provide `output_dir`. |
| `name` | str | yes | none | no | Certificate short name and output file stem. |
| `output_dir` | path | no | `<base_dir>/certs/<name>` | no | Directory containing source files and receiving the bundle. |
| `owner` | str | no | none | no | Owner applied to the bundle file. |
| `group` | str | no | none | no | Group applied to the bundle file. |
| `mode` | str | no | `0600` | no | Bundle file mode. |
| `force` | bool | no | `false` | no | Rewrites the bundle even if content matches. |

Returns:

- `changed`
- `path`

## `ca_fritzbox_deploy`

Uploads a FritzBox PEM import bundle to FRITZ!OS.

Behavior:

- Reads `<output_dir>/<name>-fritzbox.pem` unless `bundle_path` is set.
- Validates that the bundle contains at least one PEM certificate and an
  unencrypted RSA private key.
- Reads the current certificate from the FRITZ!Box HTTPS endpoint and compares
  its SHA-256 fingerprint with the desired leaf certificate.
- Requires an HTTPS `base_url` for the default idempotent comparison.
- Logs in through `login_sid.lua` with the FRITZ!OS challenge-response flow,
  including PBKDF2 challenges and the legacy MD5 response.
- Uploads the bundle to `/cgi-bin/firmwarecfg` as multipart form data.
- Logs out after the upload attempt when a session was acquired.
- Skips login and upload when the current HTTPS certificate already matches.
- `force=true` skips the comparison and always uploads.

Parameters:

| Parameter | Type | Required | Default | Secret | Behavior |
| --- | --- | --- | --- | --- | --- |
| `base_dir` | path | yes | none | no | Base CA directory. Used to derive default `output_dir`. |
| `certificate` | dict | no | `{}` | yes | Certificate model dictionary. May provide `output_dir` and `fritzbox_deploy`. |
| `deploy` | dict | no | `{}` | yes | Deployment settings. Values override `certificate.fritzbox_deploy`. |
| `name` | str | yes | none | no | Certificate short name and output file stem. |
| `output_dir` | path | no | `<base_dir>/certs/<name>` | no | Directory containing the FritzBox bundle. |
| `bundle_path` | path | no | `<output_dir>/<name>-fritzbox.pem` | no | Explicit FritzBox bundle path. |
| `base_url` | str | yes | none | no | FRITZ!Box base URL, for example `https://fritz.box`. HTTPS is required unless `force=true`. |
| `username` | str | yes | none | no | FRITZ!OS user name. |
| `password` | str | yes | none | yes | FRITZ!OS password. |
| `timeout` | int | no | `30` | no | HTTP request timeout in seconds. |
| `validate_certs` | bool | no | `true` | no | Validate HTTPS certificates for the FRITZ!Box connection. |
| `force` | bool | no | `false` | no | Upload the bundle without comparing the current FRITZ!Box HTTPS certificate. |

Returns:

- `changed`
- `path`

## `ca_dhparam`

Manages Diffie-Hellman parameter files.

Behavior:

- Generates PEM PKCS#3 DH parameters.
- Defaults to `<base_dir>/dhparams.pem` unless `path` is supplied.
- Compares the existing parameter size for idempotence.
- DH parameter generation is CPU-expensive, especially at 4096 bits.

Parameters:

| Parameter | Type | Required | Default | Secret | Behavior |
| --- | --- | --- | --- | --- | --- |
| `base_dir` | path | yes | none | no | Base directory used for the default output path. |
| `path` | path | no | `<base_dir>/dhparams.pem` | no | Explicit output path. |
| `size` | int | no | `4096` | no | DH prime size in bits. |
| `owner` | str | no | none | no | Owner applied to the parameter file. |
| `group` | str | no | none | no | Group applied to the parameter file. |
| `mode` | str | no | `0644` | no | Parameter file mode. |
| `force` | bool | no | `false` | no | Regenerates parameters even if the existing size matches. |

Returns:

- `changed`
- `path`
