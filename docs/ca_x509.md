# ca_x509

Internal X.509 implementation helpers.

`module_utils/ca_x509.py` is not an Ansible module. It contains the shared
X.509 implementation used by CA authority, certificate, CRL, and PKCS#12
modules.

Generic time parsing, serial formatting, and renewal policy handling live in
`ca_time`, `ca_serial`, and `ca_renewal`; `ca_x509` consumes those helpers
instead of carrying its own copies.

## Public Helpers

| Helper | Purpose |
| --- | --- |
| `ca_authority_argument_spec(defaults=None)` | Builds the Ansible argument spec for `ca_authority`. |
| `certificate_params(params, default_formats=None)` | Merges a certificate dictionary with explicit module parameters and derives the signer passphrase field. |
| `ensure_x509(params, signed, authority=False, manage_directory=False, manage_chain=False)` | Ensures key, CSR, certificate, optional DER, optional text, and optional chain copy. |
| `normalize_formats(formats)` | Lowercases format names and rejects string input. |
| `load_private_key(path, passphrase)` | Loads a PEM private key. |
| `load_certificate(path)` | Loads one PEM or DER certificate. |
| `load_certificates(path)` | Loads one or more certificates from PEM blocks or one DER certificate. |
| `subject_from_params(params)` | Builds an X.509 subject from `subject_ordered` or subject defaults plus `common_name`. |
| `digest_algorithm(name)` | Returns a cryptography digest object. |
| `signature_algorithm(private_key, digest)` | Returns the digest or `None` for Ed25519 and Ed448. |
| `sanitize_error(exc, params=None)` | Re-export from `ca_file`. |

## Renewal And Rekey

`ensure_x509` computes renewal before it touches the key or certificate files.
The policy accepts:

| Key | Default | Behavior |
| --- | --- | --- |
| `warn_before_days` | `0` | Adds warning state to inventory only. |
| `renew_before_days` | `0` | Reissues when remaining validity is inside this window. |
| `renew_at` | empty | Reissues once at or after the timestamp for certificates issued before it. |
| `rekey` | `false` | Generates a new private key when renewal is due. |

Same-key renewal keeps the existing private key and creates a new certificate
serial. Re-key renewal archives the old encrypted private key, generates a new
key, and issues a new certificate. A key type or size change is treated as a
re-key as well.

Before replacing a certificate generation, public artifacts are archived below
`<base_dir>/archive`. Private keys are archived only when the key is replaced.

## Supported Algorithms

Key type aliases:

- `RSA`
- `ECDSA`
- `EC`
- `P256`
- `P384`
- `ECDSA_P256`
- `ECDSA_P384`
- `prime256v1`
- `secp256r1`
- `secp384r1`
- `Ed25519`
- `Ed448`

RSA uses `key_size` as bit size. Generic `ECDSA` or `EC` uses `key_size: 256`
for P-256 and `key_size: 384` for P-384. Ed25519 and Ed448 ignore `key_size`
and `digest`.

Supported digests for RSA and ECDSA:

- `sha1`
- `sha224`
- `sha256`
- `sha384`
- `sha512`

Basic Constraints tokens:

- `CA:TRUE`
- `CA:FALSE`
- `pathlen:<n>`

Key Usage tokens:

- `digitalSignature`
- `nonRepudiation`
- `contentCommitment`
- `keyEncipherment`
- `dataEncipherment`
- `keyAgreement`
- `keyCertSign`
- `cRLSign`
- `encipherOnly`

- `serverAuth`
- `clientAuth`
- `codeSigning`
- `emailProtection`
- `timeStamping`
- `OCSPSigning`
- `smartcardLogon`

Dotted OIDs are accepted as EKU values.

SAN syntaxes:

- `DNS:<name>`
- `IP:<address>`

- `URI:<uri>`
- `RID:<oid>`

Raw extension syntaxes:

- `ASN1:BMPSTRING:<value>`
- `ASN1:UTF8String:<value>`
- `ASN1:FORMAT:HEX,OCTETSTRING:<hex>`
- `DER:<hex>`

## Derived Paths

For authorities:

- private key: `<base_dir>/private/<name>-ca.key`
- CSR: `<base_dir>/csr/<name>-ca.csr`
- PEM certificate: `<base_dir>/ca/<name>-ca.pem`
- DER certificate: `<base_dir>/ca/<name>-ca.der`
- text certificate: `<base_dir>/ca/<name>-ca.txt`

For certificates:

- output directory: `<base_dir>/certs/<name>` unless overridden
- private key: `<output_dir>/<name>.key`
- CSR: `<base_dir>/csr/<name>.csr`
- PEM certificate: `<output_dir>/<name>.pem`
- DER certificate: `<output_dir>/<name>.der`
- text certificate: `<output_dir>/<name>.txt`
- issuer chain copy: `<output_dir>/<name>-chain.pem`

AIA defaults to `<base_url>/aia/<issuer>-ca.der`.
CDP defaults to `<base_url>/crl/<issuer>-ca.crl`.

## Idempotence

`ensure_x509` compares existing keys, CSRs, certificates, DER exports, text
exports, and chain copies with desired state. Certificates are reissued when
the existing certificate is expired, the subject differs, the issuer differs,
the public key differs, or the extension set differs.

## Internal Example

```python
params = certificate_params(module_params, default_formats=["pem", "der", "txt"])
result = ensure_x509(params, signed=True, manage_directory=True, manage_chain=True)
```
